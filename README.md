## 🔌 API для разработчиков

Начиная с актуальной версии, `fcChat` предоставляет публичное API для интеграции других плагинов.

### Что можно делать через API

- Отправлять `broadcast` / `private` / `reply` сообщения
- Переключать каналы игрокам и читать текущий канал
- Чистить чат программно
- Подписываться на события `FcChatBroadcastEvent`, `FcChatPrivateMessageEvent`, `FcChatChannelSwitchEvent`
- Добавлять собственные контекстные плейсхолдеры (`{ctx:key}` / `{context:key}`)
- Настраивать аудиторию broadcast через `AudienceResolver`
- Регистрировать пользовательские действия через `ActionHandler`

### Получение API

```java
import fc.plugins.fcchat.api.FcChatApi;
import fc.plugins.fcchat.api.FcChatApiProvider;
import org.bukkit.Bukkit;
import org.bukkit.plugin.RegisteredServiceProvider;

// Вариант 1: через ServicesManager (рекомендуется)
RegisteredServiceProvider<FcChatApi> reg = Bukkit.getServicesManager().getRegistration(FcChatApi.class);
FcChatApi api = reg != null ? reg.getProvider() : null;

// Вариант 2: через провайдер
FcChatApi directApi = FcChatApiProvider.get();
```

### Пример: отправка broadcast

```java
boolean sent = api.sendBroadcast(sender, "Серверный ивент начался!");
if (!sent) {
    sender.sendMessage("Broadcast отменен другим плагином");
}
```

### Пример: свой контекстный плейсхолдер

```java
api.registerContextPlaceholder("rank_short", (sender, viewer, rawMessage) -> {
    // Возвращайте любую динамику для {ctx:rank_short}
    return "VIP";
});

// Теперь в форматах fcChat можно использовать:
// {ctx:rank_short} или {context:rank_short}
```

### Пример: фильтр аудитории broadcast

```java
import fc.plugins.fcchat.api.audience.AudienceResolver;
import org.bukkit.entity.Player;

api.registerAudienceResolver(new AudienceResolver() {
    @Override
    public String getId() {
        return "my_world_filter";
    }

    @Override
    public int getPriority() {
        return 100;
    }

    @Override
    public java.util.Set<Player> resolveRecipients(org.bukkit.command.CommandSender sender, String message, java.util.Set<Player> currentRecipients) {
        java.util.Set<Player> result = new java.util.HashSet<>();
        for (Player p : currentRecipients) {
            if (p.getWorld().getName().equalsIgnoreCase("world")) {
                result.add(p);
            }
        }
        return result;
    }
});
```

### Пример: отмена broadcast через событие

```java
import fc.plugins.fcchat.api.event.FcChatBroadcastEvent;
import org.bukkit.event.EventHandler;
import org.bukkit.event.Listener;

public class MyListener implements Listener {
    @EventHandler
    public void onBroadcast(FcChatBroadcastEvent event) {
        if (event.getMessage().toLowerCase().contains("запрещено")) {
            event.setCancelled(true);
        }
    }
}
```

### Важно

- Рекомендуется проверять `api != null` перед использованием
- Для ранней инициализации можно слушать событие `FcChatApiReadyEvent`
- Все API-классы находятся в пакете `fc.plugins.fcchat.api`
